## CORE WEB VITALS TARGETS

### Largest Contentful Paint (LCP)
Target: < 2.5 seconds
Optimization Strategies:
- Preload critical above-fold images
- Optimize server response time
- Use CDN for static assets
- Implement lazy loading for below-fold content
- Compress and optimize images (WebP format)

### First Input Delay (FID) / Interaction to Next Paint (INP)
Target: < 100 milliseconds
Optimization Strategies:
- Minimize JavaScript bundle size
- Code splitting for route-based loading
- Defer non-critical JavaScript
- Use web workers for heavy computations
- Optimize event handlers

### Cumulative Layout Shift (CLS)
Target: < 0.1
Optimization Strategies:
- Reserve space for dynamic content
- Define image dimensions (width/height)
- Avoid inserting content above existing content
- Use CSS aspect-ratio for media
- Preload web fonts with fallbacks

## CACHING STRATEGIES

### Browser Caching
Static Assets:
- JavaScript: 1 year (fingerprinted)
- CSS: 1 year (fingerprinted)
- Images: 1 year (fingerprinted)
- Fonts: 1 year (fingerprinted)

Dynamic Content:
- HTML: No cache (revalidate on request)
- API data: Short cache (1-5 minutes)
- User data: No cache (personalized)

### Service Worker Caching
Cache-First Strategy:
- Static assets (JS, CSS, fonts)
- Product images
- Icon assets

Network-First Strategy:
- API responses
- Product data
- User session data

### CDN Configuration
Edge Caching:
- Static assets: Global edge distribution
- Images: WebP conversion at edge
- HTML: Origin caching with short TTL

## RESOURCE LOADING OPTIMIZATION

### Critical Resource Prioritization
High Priority:
1. HTML document
2. Critical CSS (inlined)
3. Hero image (preload)
4. Main JavaScript bundle

Medium Priority:
1. Product images (lazy load)
2. Non-critical CSS
3. Analytics scripts

Low Priority:
1. Third-party widgets
2. Below-fold content
3. Prefetched pages

### Preloading Strategy
```html
<link rel="preload" href="/fonts/inter.woff2" as="font" type="font/woff2" crossorigin>
<link rel="preload" href="/cbn-logo.png" as="image">
<link rel="preconnect" href="https://fonts.googleapis.com">
```

### Code Splitting
Route-based Splitting:
- /products → products.chunk.js
- /battery-selector → selector.chunk.js
- /battery-guide → guide.chunk.js
- /contact → contact.chunk.js

Component-based Splitting:
- Product modals: Lazy loaded on interaction
- Image galleries: Lazy loaded on scroll
- Forms: Lazy loaded on route access
